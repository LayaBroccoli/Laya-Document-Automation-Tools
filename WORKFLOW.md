# 文档自动生成工作流

> 一句话生成带图片和动态图的完整文档

## 工作流架构

```
用户输入
    ↓
意图识别 → 任务类型（教程/API/示例）
    ↓
查询MCP → 获取API、示例、组件属性
    ↓
生成计划 → 拆解步骤、确定截图点
    ↓
执行操作 → 在IDE中实际操作验证
    ↓
采集素材 → 截图、录GIF
    ↓
组装文档 → 按模板生成最终文档
```

## 任务类型

| 类型 | 触发词示例 | 输出结构 |
|------|------------|----------|
| 教程类 | "如何创建XXX"、"XXX入门" | 目标→步骤→成果 |
| API类 | "XXX组件文档"、"XXX属性" | 概念→属性→示例 |
| 实战类 | "实现XXX功能"、"XXX怎么做" | 问题→方案→代码 |

## 步骤拆解规则

### 1. 每个独立操作 = 1个步骤 + 1-2张截图

```
❌ 不好：打开IDE，创建项目，设置属性，保存
✅ 好：
  步骤1：创建项目
  步骤2：设置属性
  步骤3：保存场景
```

### 2. 截图时机

- **操作前截图**：展示初始状态
- **操作后截图**：展示结果
- **动图时机**：拖拽、连续操作、运行效果

### 3. 截图命名规范

```
{章节}-{序号}.png          静态图
{章节}-{序号}.gif          动图
{章节}-{序号}-{标注}.png   带红框标注
```

```markdown

![](img/1-1.png)

（图1-1）{图注}

> {注意事项或解释}

## 这一步的成果

{总结本步完成的内容}

```

```markdown

{概述：是什么、有什么用}

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| {表格列举} |

{代码示例 + 效果图}

## 常见用法

{分场景说明}

## 关键概念

{深入讲解1-2个核心概念}
```

## 工作流命令

```bash
# 生成文档
node workflow.js "写一个如何使用PhysicsCollider组件的文档"

# 指定输出目录
node workflow.js "制作滚动背景教程" --output docs/scrolling/

# 只生成不执行（预览计划）
node workflow.js "RigidBody组件入门" --dry-run

# 指定截图范围
node workflow.js "动画系统教程" --chapters 1-3
```

## MCP查询策略

| 需求 | MCP工具 | 参数 |
|------|---------|------|
| 查组件 | get_schema_by_name | 组件名 |
| 查API | get_api_detail | API名 |
| 查示例 | get_examples | 功能描述 |
| 搜索 | query_api | 关键词 |

## 截图策略

### 静态截图

```javascript
// 截取整个IDE窗口
s.js win output.png

// 截取区域
s.js win output.png --crop 0,0,1334,750

// 截取+标注
s.js win output.png --mark 100,200,80,20
```

### 动图录制

```javascript
// 录制60帧，15fps
s.js gif output.gif --frames 60 --fps 15

// 从游戏target录制
s.js gif output.gif --game --frames 45
```

### 智能截图

```javascript
// 查找元素并截图标注
s.js find "增加组件"
s.js win output.png --mark {返回的坐标}
```

## 质量检查规则

### 内容检查

- [ ] 每个步骤都有操作描述
- [ ] 代码可运行
- [ ] 有"这一步的成果"总结
- [ ] 有"下一步"引出

### 图片检查

- [ ] 关键步骤都有截图
- [ ] 图片清晰无多余内容
- [ ] 动图长度适中（2-5秒）
- [ ] 标注位置准确

### 格式检查

- [ ] 标题编号正确（## 1、XXX）
- [ ] 代码块语言正确
- [ ] 图注格式正确（（图X-X)）
- [ ] 无emoji

## 示例工作流

### 输入

```
"写一个如何创建PhysicsCollider并设置碰撞组的文档"
```

### 执行流程

1. **意图识别** → 教程类
2. **MCP查询** → `get_schema_by_name("PhysicsCollider")`
3. **生成计划** →
   - 步骤1：创建3D物体
   - 步骤2：添加PhysicsCollider组件
   - 步骤3：设置碰撞组
   - 步骤4：运行验证
4. **执行** → 在IDE中实际操作
5. **截图** → 每步截图，录拖拽动图
6. **生成** → 按模板输出文档

### 输出

```markdown
# 添加碰撞检测组件

{内容}
```
